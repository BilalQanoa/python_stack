Flask Routing - My First Step in Backend 🚀
This is my first project using Flask, where I focused on understanding how servers handle different URLs and how to make routes dynamic. No more static pages—it's time for the server to actually "do" something!

💡 What did I build?
I implemented 5 core features to master Flask routing:

The Classic "Hello World": Just to make sure the server is alive and kicking.

Static Routes: Created a /champion route because why not? We all need a little motivation.

Dynamic Greetings: I made a route that greets users by name. I used.capitalize() to make sure the names look professional even if the user is lazy with their typing.

The Repeater: This was fun! I learned how to handle integers in URLs using <int:num>. It repeats a word as many times as you want. It's a great way to see how Flask handles data types.

Custom 404 Page: I hate the default "Not Found" pages, so I created my own friendly message to guide users back if they get lost.

🛠️ Tools Used
Python (The brain)

Flask (The bridge)

F-strings (For clean and readable string formatting)

🧠 Lessons Learned
The biggest takeaway for me was understanding that HTTP is stateless. Every request is a brand-new conversation. I also learned that small details—like handling errors and formatting strings—make a huge difference in the final result.
